# Security Policy

## Reporting a Vulnerability

Use GitHub's [private vulnerability reporting](https://github.com/Bizarre-Industries/Helling/security/advisories/new)
or email [skalghazali@gmail.com](mailto:skalghazali@gmail.com).

Include:

- Affected component
- Reproduction steps
- Impact assessment
- Suggested mitigation if available

## Disclosure Process

- Initial triage acknowledgement target: 72 hours
- Fix timeline depends on severity and exploitability
- Coordinated disclosure is preferred

## Scope

In scope:

- Authentication and authorization flaws
- Privilege escalation paths
- Sensitive data exposure
- Supply-chain integrity issues in release or CI artifacts

Out of scope:

- Social engineering attacks
- Vulnerabilities in unsupported forks
