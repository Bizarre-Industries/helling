## Headline

**The WebUI is mostly built. The spec it was built against has been quietly abandoned.**

51 findings across 9 areas + 5 reframings. Stack drift (F-36) is upstream of half of v0.1's findings. Auth-store violates `docs/spec/auth.md` §2.2 (F-37). Login is theatre (F-38). Zero error boundaries (F-39).

## Top six actions

1. **Decide the stack.** F-36. Doc-vs-code split on antd is the most expensive bug in the repo. Resolved by ADR-051 (commit to spec).
2. **Fix auth-store.** F-37. Access tokens in localStorage violate `docs/spec/auth.md` §2.2 and are XSS-exfiltratable.
3. **Add one root error boundary.** F-39. Cheapest blast-radius reduction in the codebase.
4. **Decide on the data layer.** F-01. Components own mock state directly; every new page makes the eventual API migration worse.
5. **Split `pages.jsx` + `pages2.jsx`.** F-05. 8,599 lines in two files.
6. **A11y pass.** F-25/F-26/F-27/F-28. Zero `aria-*`, zero `role=`. WCAG 2.1 AA spec violation.

## Reframings (v0.2)

- **R-01 → F-03**: SSE not `refetchInterval`. OpenAPI ships `eventsSse`; build `useEventsStream()` consumer.
- **R-02 → F-06/F-14/F-23/F-25/F-26**: gated on F-36. Pro-components absorb these natively if spec wins.
- **R-03 → F-22**: viewport meta `width=1440` is non-standard, breaks pinch-zoom. Use `width=device-width` + CSS gate.
- **R-04 → F-29**: verify Unbounded is actually downloaded before claiming savings. Code-splitting + Big Shoulders subset still real.
- **R-05 → F-04**: sequence after F-38 (wire login first, then plug error states into a real auth boundary).
