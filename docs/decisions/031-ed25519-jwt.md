# ADR-031: Ed25519 for JWT signing

> Status: Accepted (2026-04-19)

## Context

Current docs permit multiple JWT signing algorithms. That flexibility increases implementation complexity and can create interoperability drift between components.

## Decision

Use EdDSA with Ed25519 as the only JWT signing algorithm for Helling v0.1.

Requirements:

- Access and refresh tokens are signed with Ed25519
- Verification rejects tokens signed with other algorithms
- Key rotation is supported with overlapping verification window
- Public keys are exposed through JWKS-compatible metadata when needed

## Consequences

- Single signing profile across CLI, proxy, and API consumers
- Smaller key and signature management surface
- Simpler security review and test coverage
- Migration path for older token algorithms must be explicit if introduced in future versions
